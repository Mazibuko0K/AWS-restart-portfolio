# Database Operations Labs – Summary

## Overview

These labs focused on practicing core relational database operations using a MySQL client hosted on an EC2 instance. The work involved connecting to a remote environment, interacting with databases, and performing common data definition and data manipulation tasks.

---

## Lab 1: Database and Table Management

This lab covered foundational database structure operations. I connected to a command host, created a new database, and defined a table with a structured schema. I explored how to inspect database objects and corrected schema issues using modification commands. The lab also introduced safe deletion practices for database objects.

**Key Skills Practiced:**

* Creating databases and tables
* Viewing database structures
* Modifying table schemas
* Deleting tables and databases

---

## Lab 2: Data Manipulation and Importing

This lab focused on working with the data inside tables. I inserted records, updated existing data, and removed entries from a table. I also practiced bulk data loading using an SQL script file, which demonstrated a more efficient way to populate databases.

**Key Skills Practiced:**

* Inserting and querying data
* Updating table records
* Deleting data safely
* Importing data from SQL files

---

## Key Takeaways

* SQL provides structured commands for both managing database schemas and manipulating data
* Care must be taken when using destructive operations like `DELETE` and `DROP`
* Automation through SQL scripts significantly improves efficiency when handling large datasets
* Understanding both structure (DDL) and data (DML) operations is essential for database management
* Obviously these are just 2 samples, as there are more labs that went deeper into databases and SQL
