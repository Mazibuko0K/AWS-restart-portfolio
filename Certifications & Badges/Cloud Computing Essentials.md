# Cloud Computing Essentials

The goal of this project was to implement core AWS services (EC2, S3, VPC, and IAM) to build a foundational cloud infrastructure, demonstrating the shift from traditional on-premises deployments to scalable, on-demand cloud resources.

## Theory & the "Why"

**On-Demand Resources & Pay-as-You-Go Model (Flexible Infrastructure):**
This is the primary reason for adopting cloud computing. AWS provides virtually unlimited compute, storage, and networking resources that can be provisioned in minutes rather than weeks, eliminating the need for upfront hardware investments and long procurement cycles. This enables use cases such as:

* Rapid application development and testing environments
* Scalable web applications and APIs
* Data storage and backup solutions
* Global content delivery and low-latency access

The customer did not want to invest in physical data centers or maintain expensive hardware, so a cloud-first approach works well here especially when paired with automation tools like CloudFormation or Terraform. Due to the elastic nature of cloud resources, you get flexibility: scale up during peak demand and scale down during quiet periods, paying only for what you use. AWS also provides global infrastructure with multiple regions and availability zones, enabling geographic redundancy and disaster recovery.

In essence, you move to cloud computing when you need agility, cost optimization, and the ability to focus on your applications rather than managing underlying infrastructure.
